# 🛡️ Руководство по гигиене Git для проектов на Python

Этот проект содержит настройки для безопасной и чистой работы с Git. Ниже приведена инструкция по использованию `.gitignore` и управлению секретами.

## ⚙️ Настройка .gitignore

В корне проекта создан файл `.gitignore`, который блокирует попадание лишнего мусора в репозиторий.

### Что игнорируется:
*   **Секреты:** `.env` — ваши пароли и API-ключи.
*   **Окружение:** `venv/` — тяжелые папки библиотек (устанавливаются локально через `pip install -r requirements.txt`).
*   **Кэш:** `__pycache__` — временные файлы компиляции Python.
*   **Настройки IDE:** `.idea/` и `.vscode/` — ваши личные предпочтения редактора.

---

## 🔑 Работа с переменными окружения (.env)

Мы не храним секреты в коде. Все настройки находятся в файле `.env`, который добавлен в исключения.

### Как развернуть проект коллеге:
1. Найдите файл-шаблон **`.env.example`**.
2. Скопируйте его и переименуйте в **`.env`**.
3. Впишите свои локальные доступы и ключи.

### Как обновить шаблон (для разработчика):
Если вы добавили новую переменную в `.env`, обновите шаблон командой в терминале:

*   **macOS / Linux:**
    ```bash
    sed 's/=.*/=/' .env > .env.example
    ```
*   **Windows (PowerShell):**
    ```powershell
    (Get-Content .env) -replace '=.*', '=' | Out-File .env.example -Encoding utf8
    ```

---

## 🛠️ Если файл попал в Git по ошибке
Если вы случайно закоммитили `.env` или `venv/`, удалите их из индекса (они останутся у вас на диске, но исчезнут из Git):

```bash
git rm -r --cached .env
git rm -r --cached venv/
git commit -m "Задача: удалить конфиденциальные файлы из индекса."
